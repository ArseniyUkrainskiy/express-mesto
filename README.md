## Проект Mesto бэкенд

Сервер на express. :floppy_disk: Он умеет проверять токены, регистрировать и авторизовывать пользователей, сохранять и отдавать карточки, запоминать, когда кто-то поставил лайк или передумал и убрал его.

:dart: Цель работы: научиться работать с базами данных, разбираться в безопасности и тестировании, разворачивать бэкенд на удалённой машине.

* Node.js.
* Тестирование сервера: `Postman`.
* Фреймворк для разработки серверного кода — Express.
* middlewares/ Сборка пакетов: body-parser/ .
* CORS. Обработка ошибок.
* MongoDB.
* Роутеры, контроллеры, схемы и модели.
* `ESLint`/ Airbnb/..
* Аутентификация и авторизация, защита роутов.
* Централизованная обработка ошибок (400, 401, 403, 404, 409, 500).
* Валидация приходящих на сервер данных, библиотеки Joi и celebrate. + регулярные выражения /regExp/
* Безопасность. XSS, JWT, CSRF, DDоSу :bricks: (модуль Helmet, escape-html, LocalStorage, express-rate-limit, audit, файл .env). :stop_sign:

### Директории

`/routes` — папка с файлами роутера  
`/controllers` — папка с файлами контроллеров пользователя и карточки   
`/models` — папка с файлами описания схем пользователя и карточки  
`/middlewares` — папка с мидлверами аутентификации и централизованной обработкой ошибок  
`/errors` — папка с классами ошибок  
  
Остальные директории вспомогательные, создаются при необходимости разработчиком

### Запуск проекта

:small_blue_diamond: npm run start — запускает сервер   
:small_blue_diamond: npm run dev — запускает сервер с hot-reload  
:small_blue_diamond: npm run lint--fix — запускает проверку и автоматическое исправление согласно настройкам
